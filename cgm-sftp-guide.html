SFTP Onboarding Guide

How CGM Virtuox delivers the B2B patient AR invoice feed to AgentCollect. Standard, low-lift, no new enterprise tooling required.

TL;DR. A nightly CSV export from your existing AR system (Aprima, eMDs, or the current data warehouse) dropped into an SFTP folder. We pick it up every morning. That's the whole integration. It works with the stack CGM already runs; no new SAP module, no new middleware, no bespoke engineering on CGM's side.

What We're Building, In One Paragraph

CGM's existing AR database already holds every invoice we need. On a nightly schedule (typically 2:00 AM local), the database exports a filtered CSV ("invoices below $20,000, aged 90+ days past due, Virtuox division") and drops it onto an SFTP endpoint. AgentCollect's ingestion service connects to that endpoint on a read-only SSH key, pulls the file, and loads it into the agent's case queue. Payments and dispositions flow back the other way: AgentCollect writes an updated CSV back to the same SFTP in a /results folder; CGM reads it on its next scheduled run. Two CSVs, two folders, one SSH key pair. That's it.

Why This Is Not a Hard Integration

  • Every major EHR / practice-management stack supports scheduled CSV exports. Aprima, eMDs, Epic, Cerner, athenahealth, NextGen all do this natively. CGM IT has almost certainly set up similar feeds for other vendors (eligibility, statements, collections) many times.
  • SFTP is the healthcare industry default. It predates modern APIs, it's HIPAA-compatible out of the box, and it's what claim clearinghouses, lockbox services, and third-party collectors have used for decades. Every healthcare IT team has done this before.
  • We do not require a new ERP, middleware, or enterprise service bus integration. Specifically: no SAP module, no Mulesoft, no Informatica, no custom REST endpoints. If Aprima can run a scheduled report and write it to disk, we're done.
  • Read-only SSH keys, not passwords. CGM generates an SSH key pair, gives us the public key and the SFTP host; we use the private key (stored in our HashiCorp Vault) to pull. No shared passwords, no rotating secrets.

Typical Setup, Step by Step

1
CGM IT provisions the SFTP endpoint
Either a new VM running OpenSSH (30 min setup), a managed SFTP service (AWS Transfer Family, Azure Blob SFTP, etc.), or a slot on an existing shared SFTP server CGM already operates. Any of these work. Typical time: 1–4 hours.
2
CGM creates an SSH key pair for AgentCollect
Standard ssh-keygen -t ed25519. Public key installed on the SFTP server; private key sent to AgentCollect via secure channel (1Password shared vault, or direct email to j@agentcollect.com if acceptable to CGM security).
3
CGM configures the nightly AR export from Aprima
A scheduled report in the AR system (Aprima, eMDs, or the Virtuox data warehouse) filters: balance < 20000 AND days_past_due >= 90 AND division = 'Virtuox'. Output format is a CSV written to /incoming/ on the SFTP endpoint every night. Standard Aprima / eMDs functionality; no custom code.
4
AgentCollect ingests on its side
Our side is already built. Pulls /incoming/*.csv, validates, loads to the case queue, launches the voice agent. Returns payments and dispositions to /results/*.csv. Tested on previous healthcare clients.
5
Go-live
We run a 2-day read-only dry run on one day of data to verify format. Then we flip live. Total calendar time from SFTP endpoint ready to first patient call: typically 7–10 business days on our side.

Security & Compliance

  • HIPAA BAA signed between Respaid Inc. (d/b/a AgentCollect) and CompuGroup Medical before any PHI flows. Template attached to the LOI countersignature package.
  • SFTP over SSH (port 22) with ed25519 or RSA-4096 keys. No password authentication.
  • Key storage: AgentCollect side uses HashiCorp Vault with automatic rotation every 90 days.
  • Audit trail: every pull and push logged with timestamp, file hash, and line count. Logs exportable to CGM on request.
  • TCPA: any DNC / consent flags in the CSV are honored before dial. AgentCollect maintains its own internal DNC list that merges with CGM's.
  • Data residency: US region only. AWS us-east-1 / us-west-2. No data leaves the US.
